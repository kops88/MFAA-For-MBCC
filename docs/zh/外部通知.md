# 外部通知（用户填写指南）

本页只讲“**怎么填**”。填写后可用“发送测试”验证。

---

## 钉钉（DingTalk）

**填写方式**
- **Bot Token**：可直接粘贴完整 Webhook（例如 `https://oapi.dingtalk.com/robot/send?access_token=xxx`），会自动提取 `access_token`
- **Secret**：机器人“加签密钥”。未开启加签可留空

**实际行为**
- 有 `Secret` → 使用签名请求  
- 无 `Secret` → 走普通 Webhook

---

## 邮箱（Email）

**填写方式**
- **Bot Token**：你的邮箱地址（如 `xxx@qq.com`）
- **Email Secret**：邮箱授权码/应用密码（不是登录密码）

**实际行为**
- 会根据邮箱域名自动匹配 SMTP 配置

---

## 飞书（Lark）

飞书有两种填写方式：

### 方式 A：Webhook（极简，推荐）
- **Webhook URL**：直接填飞书机器人 Webhook

**行为**：只要填了 Webhook URL，就会**优先使用 Webhook**，不需要填其它字段。

### 方式 B：签名模式（普通）
- **Lark ID**：机器人 ID  
- **Bot Token**：机器人密钥

**行为**：Webhook 为空时才会走签名模式。

---

## WxPusher

WxPusher 有两种填写方式：

### 方式 A：普通推送
- **Bot Token**：AppToken  
- **UID**：用户 UID

### 方式 B：极简推送
- **Bot Token**：留空  
- **UID**：填写 SPT

---

## Telegram

**填写方式**
- **Chat ID**：聊天或群组 ID  
- **Bot Token**：机器人 Token

---

## Discord Bot

**填写方式**
- **Channel ID**：频道 ID  
- **Bot Token**：机器人 Token

---

## Discord Webhook

**填写方式**
- **Webhook URL**：Webhook 链接  
- **Webhook Name**：显示名称（可留空）

---

## OneBot

**填写方式**
- **Server**：OneBot 服务地址（如 `http://127.0.0.1:5700`）  
- **Key**：鉴权 Key（若服务需要）  
- **User**：接收消息的 QQ 号

**说明**
- 该渠道目前未接入统一发送流程，需要自行接入后才能使用。

---

## SMTP

**填写方式**
- **SMTP Server**：服务器地址（如 `smtp.qq.com`）  
- **SMTP Port**：端口（常见 465/587/25）  
- **SMTP SSL**：SSL 开关  
- **SMTP Auth**：是否需要账号密码  
- **SMTP User**：账号（仅在 Auth 开启时填写）  
- **SMTP Password**：密码/授权码（仅在 Auth 开启时填写）  
- **SMTP From**：发件人地址  
- **SMTP To**：收件人地址

---

## Qmsg

**填写方式**
- **Server**：Qmsg 服务地址  
- **Key**：Qmsg Key  
- **User**：接收者 QQ  
- **Bot**：机器人 QQ（可留空）

---

## Server酱

**填写方式**
- **Send Key**：Server 酱 SendKey

**实际行为**
- 支持 `sctp` 形式的 SendKey

---

## 通用 Webhook

**填写方式**
- **Webhook URL**：完整 `http/https` 地址  
- **Content Type**：默认 `application/json`  
- **Payload Template**：模板中使用 `{message}` 作为占位符

**示例模板**
```
{"text":"{message}"}
```

---

## 常见问题

- **收不到消息**：检查字段是否填完整、网络是否可访问  
- **Webhook 报错**：必须是完整 `http/https` URL  
- **邮箱失败**：多数邮箱要求授权码而非登录密码
